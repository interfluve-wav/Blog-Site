Bonk!

Desktop app for DJs: edit music metadata, manage playlists, and stay in sync with Rekordbox 6/7 — or work file-first with folder import and Quick Tag. Built as Electron + React with a strict split: the UI never touches Node; everything system-related goes through window.electronAPI.

What you can do

  • Rekordbox — Import/export encrypted master.db via a Python bridge (rekordbox_bridge.py + vendored pyrekordbox).
  • Library mode — Virtualized track table, playlist tree, smart playlists, search index, column layout, track editor panel.
  • Folder import — Scan folders, read tags, optional enrichment flows; file-scoped writes where those modes apply.
  • Quick Tag — Single-folder fast tagging with Rust-backed playback and waveform.
  • Analysis — Key (KeyFinder CLI), BPM (aubio), waveforms (Rust + cache), optional batch wizards.
  • AutoTag / Find Tags — Multi-provider metadata lookup (Spotify, Discogs, MusicBrainz, etc.) orchestrated from the main process.
  • Format tools — Transcode and tag tooling using FFmpeg and TagLib-style writers where safe.
  • Extras — Chapter Builder, Graph Playlist, analytics, download helpers, multi-library converters — entered from the shell as full views or modals.

Process architecture

Three layers: main (Node, IPC, spawns, SQLite media cache), preload (exposes one API object), renderer (React, Zustand, workers). The renderer calls electronAPI; the main process validates paths, runs Python, loads native-audio, and talks to disk.

Where state and files live

On disk (typical)

  • userData/Config/bonk-user-config.json — settings
  • userData/bonk-media-cache.sqlite — art, waveforms, cues
  • Rekordbox paths — Pioneer folders; app can import/export DB and XML per flows

In memory (renderer)

  • Full library snapshot in Zustand while a session is loaded
  • Search index (n-grams) for fast filtering
  • LRU-style caches for album art thumbnails in the UI layer

Heavy work off the UI thread

  • filterWorker.ts for large libraries
  • Main process for I/O, Python, FFmpeg, and native audio

How to run it (dev)

npm install · pip3 install -r requirements.txt (for Rekordbox features) · npm run dev. Typecheck: npm run typecheck. Ship build: npm run build:electron. See repo AGENTS.md for the full command list and file map.
